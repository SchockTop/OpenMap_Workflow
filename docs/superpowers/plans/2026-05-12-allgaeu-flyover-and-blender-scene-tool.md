## Decisions

- **Scope:** full ~45 km¬≤ polygon, **DGM1** terrain + **DOP40** (40 cm) ortho + **LoD2** buildings. (40 cm is plenty for a fly-over; ~3‚Äì4 GB DOP40 vs ~15+ GB for DOP20. Pipeline's `muc-sued-10x4` preset is already 40 km¬≤, so 45 km¬≤ is within range.) Subdiv 11 on the terrain plane (‚âà4.4 m/quad over 9 km ‚Äî fine from the air).
- **Clouds:** new `features/clouds.py` ‚Äî procedural **volumetric** cumulus deck (a big thin box with a Principled-Volume material driven by layered noise: a "coverage" control, soft bottoms, slight anvil flattening on top), placed above the terrain (Allg√§u terrain ~750‚Äì1800 m, peaks ~2000 m ‚Üí cloud base ~2200‚Äì2600 m absolute, in scene-local Z that's ~base minus anchor). Broken coverage so it doesn't block the view; tuned step size / not-too-dense for efficiency. Maybe an optional thin high cirrus layer. Operator `BLENDERTOOLS_OT_add_clouds` with props (coverage 0‚Äì1, base altitude, thickness, density, +cirrus toggle). Wired into the scene-builder + `_blender_assemble_full.py` `--enable clouds`.
- **Trees ‚Üí forest mask:** rasterize a forest mask for the AOI and feed it into `trees.py`'s `density_mask`. Best source = OSM land-use forest polygons (`OpenMap_Unifier/backend/osm_downloader.py` already produces `downloads_osm/land_use.geojson`) ‚Üí `gdal_rasterize` to a 0/1 GeoTIFF at the terrain's UV resolution ‚Üí sample as `density_mask` in the GN tree setup. Fallback if no OSM layer: an RGB "greenness" mask from the DOP (`ExG = 2G‚àíR‚àíB`, thresholded) via `gdal_calc`. Bump on-forest density, near-zero elsewhere. Also: nudge tree leaf shading toward "not obviously a card" ‚Äî a touch of translucency/back-light on the leaf material (Translucent BSDF mix or Principled `Transmission`/`Subsurface` low) so backlit forest reads as a forest mass, not flat sprites.
- **Building look from above:** `--enable buildings-textured` (DOP-on-roofs) ‚Äî already does what the user asked ("satellite mapped onto the buildings").
- **Workflow tool:** improve `BLENDERTOOLS_OT_quick_scene_from_folder` (or add `BLENDERTOOLS_OT_build_cinematic_scene` if cleaner) to be the one-stop folder‚Üíscene op: terrain + ortho + buildings + **building textures** + **forest-masked trees** + **clouds** + sky preset + camera preset + quality preset + (optional) camera-rig-from-waypoints. Accept either raw tiles *or* an already-GDAL-processed folder (`heightmap.tif` + `ortho_udim/` + `buildings.gml`/`.cityjson`) so the GDAL step can be done on machine A. Expose the key choices as operator properties (so the redo panel / N-panel works). Make sure the **N-panel "OpenMap" tab** lists this big button + the individual steps cleanly.
