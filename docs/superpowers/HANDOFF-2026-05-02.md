## Three real bugs found and fixed during integration verification

These were not in the plan — they surfaced when we ran the full pipeline.
Fixes are committed; they're worth flagging so they don't get reverted:

1. **Import cascade.** `from dop_projector import ...` worked in unit tests
   (sys.path hack) but failed silently in the production extension context
   where the module lives at `bl_ext.user_default.blender_tools.dop_projector`.
   The features ran but exited before creating the projector. Fix:
   try/except cascade in `buildings_textured.py` and `ground_shader.py`.
   Commit `965521a`.

2. **Anchor-relative DOPProjector.** Buildings are imported anchor-translated
   to scene-local coords (e.g. -50…+50), but the DOPProjector was being
   created at full UTM (e.g. 691564…). UV sampling completely missed the
   buildings. Fix: subtract `anchor_utm32n` from `bbox_utm32n` in
   `buildings_textured.apply()` before passing to `_make_roof_material`.
   Commit `965521a`.

3. **ground_shader had no terrain fallback.** trees.py finds terrain by
   name when context.terrain_obj is None; ground_shader just gave up.
   Fix: `_find_terrain_in_scene()` heuristic. Commit `965521a`.

4. **`workflows/full_pipeline.py` had a stale import path.**
   `from blender_tools.X import` — but the package directory is
   `openmap_blender_tools`. Fixed to `from openmap_blender_tools.X`.
   Commit `b60fdbf`.

## Why Tasks 14 + 17 are blocked: TWO separate visual issues

After all the Sprint 7 wiring fixes, the pipeline runs end-to-end and
the saved scene has **everything correctly wired** (verified via MCP
inspection: DOPProjector at scene-local coords, BOX projection on roof
TexImage, TexCoord.Object linked, terrain has GroundShader_Layered,
4 trees linked). But rendered images are wrong:

### Issue A — Lighting blowout (pre-existing, NOT Sprint 7)

Saved scene has:
- `CinematicSun.energy = 80.0` (default sun is ~5)
- World background color `(0.8, 0.8, 0.8)` at strength `1.0`
- Render engine `BLENDER_EEVEE` (legacy, not EEVEE Next)

That combination blows every surface to pure white regardless of materials.
Buildings render as white monopoly pieces. The DOP roof texture, the
GroundShader_Layered material, leaf-card alpha — all there, all hidden
under blowout.

**User's confirmation: "Blown out? I mean it had buildings and trees but
it didn't have a ground texture but showed the skybox of the background.
Hence the ground was already missing."** — i.e. THIS PRE-DATES SPRINT 7.

Likely sources of the blowout:
- `cinematic_preset.py` and/or `sky_presets.py` set sun energy too high
  for AgX view transform. Look at `bl_ext.user_default.blender_tools.sky_presets`,
  preset 'afternoon' specifically.
- World setup may not be applying the new world correctly; the bg color
  (0.8, 0.8, 0.8) with strength 1.0 is huge ambient flood.

### Issue B — Camera framing for small fixture region

Visual harness uses `muc-marienplatz-50m` (77×91 m terrain). But CityJSON
import brings in **5479 buildings spanning 2 km × 2 km** (whole 1km tiles
around). My `_scene_bbox_center` in `_render_slot.py` averages ALL of them,
landing scene center ~600 m away from the tiny terrain. Result:
- Wide shots: building cluster appears in corner of frame, terrain offscreen
- Close shots (`close_ground_patch`, `close_tree`, `close_seam`): aimed at
  scene center where there's nothing → empty sky

Two ways to fix per-user-choice:
- **(a)** Use a larger fixture region (`muc-sued-4x2` or similar) where
  terrain matches building extent. Slower assemble (~10 min) but visually
  correct.
- **(b)** Crop LoD2 import in `citygml_import` to the marienplatz bbox so
  buildings + terrain match scale.

User said: *"I really don't care which test region we are using. You just
picked a random region. No problem to choose something else though."*

Recommendation: **(a) — switch to `muc-sued-4x2`.** It's the showcase
region anyway (see `showcase/01_poster.png`), and aligning fixture +
showcase means one regen produces both.

```python
def _scene_bbox_center():
    for obj in bpy.data.objects:
        if obj.type == "MESH" and "Terrain" in obj.name:
            xs, ys, zs = [], [], []
            for v in obj.bound_box:
                wv = obj.matrix_world @ mathutils.Vector(v)
                xs.append(wv.x); ys.append(wv.y); zs.append(wv.z)
            return ((min(xs)+max(xs))/2, (min(ys)+max(ys))/2, max(zs))
    return (0.0, 0.0, 0.0)
```
